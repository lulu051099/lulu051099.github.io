1. 前言 / 概述

软件定位与核心价值

本系统面向职业院校机械臂实训：用自然语言或 Blockly 编排任务，经统一执行引擎与安全层驱动机械臂完成抓取、放置等操作，强调低门槛、过程可见与课堂安全。

目标用户

  • 学生：按实验指导完成自然语言或积木任务
  • 教师：组织课堂、讲解流程、查看日志
  • 实训管理员：部署、标定、网络与账号
  • 维护人员：排障与版本升级

手册阅读指南

  • 本手册目录为「操作手册」结构，不等同于作品报告论文章节
  • 建议首次阅读顺序：前言 → 使用环境 → 界面总览 → 各功能章 → 附录
  • 标定、网络、权限类操作默认由管理员在课前完成

3. 使用环境与准备工作

功能目的

在第一次使用或新开课前，确认硬件、网络、账号与工位安全条件满足，避免「能打开页面但无法执行」或安全隐患。

界面布局

  • 硬件：桌面级机械臂、工业 RGB 相机、教学主机（配置以实验室为准）
  • 软件：浏览器、后端与驱动服务（由管理员安装）
  • 权限：学生账号是否可执行任务、是否限制危险操作
  • 工位：工作区内无障碍、急停可释放、安全距离符合实验室规定

操作流程

  1. 按实验室清单完成机械臂上电与相机连接
  2. 在管理员指导下确认服务已启动、浏览器能打开控制台
  3. 确认标定数据在有效期内（若提示需标定，先完成标定再上课）
  4. 空载或低速试跑简单指令，确认日志与设备状态正常

注意事项

  • 勿在设备异常时强行重复执行；先读日志与错误码（见附录）
  • 不同教室 IP 与端口可能不同，以管理员提供的说明为准

常见问题（可选）

  • Q：页面打不开？ A：检查本机网络、防火墙与访问地址是否由管理员分配。
  • Q：提示未连接设备？ A：检查 USB/网线、驱动与设备上电顺序。

4. 界面总览与基本操作

功能目的

帮助用户快速识别「模式切换」「主工作区」「日志区」的位置，减少误操作。

界面布局

  • 模式切换：自然语言 / Blockly（二选一进入对应工作区）
  • 主工作区：任务输入、积木画布或流程编辑、预览（若有）
  • 运行控制：运行、停止、重试（以实际界面为准）
  • 日志与状态：执行过程、异常提示、连接状态

操作流程

  1. 进入系统后先确认当前模式（自然语言或 Blockly）
  2. 在主工作区完成输入或搭积木
  3. 运行前观察日志区是否提示缺参或设备未就绪
  4. 执行后根据日志与机械臂动作对照是否一致

注意事项

  • 勿在任务执行中刷新页面（除非界面说明支持断点恢复）
  • 多工位共用服务时，注意自己所在工位编号与队列提示

5. 自然语言任务（LLM）

功能目的

用自然语言描述任务，由大语言模型解析意图并生成可执行步骤，无需手写 ROS/脚本代码即可完成入门实训。

界面布局

  • 任务输入区：输入一句完整任务描述（含目标物、动作、位置关系）
  • 解析与规划区：展示步骤拆解、中间推理（可解释）
  • 执行与日志区：运行状态、异常与安全拦截提示

操作流程

  1. 确认相机与机械臂已就绪（见第 3 章）
  2. 输入任务，例如「把红色积木放到蓝色积木上」
  3. 核对系统生成的规划是否合理
  4. 运行并观察机械臂与日志；失败时按提示简化描述或重试

注意事项

  • 勿输入与课堂无关的危险诱导内容；系统有安全层，但仍须遵守实训室规范
  • 描述越具体（颜色、相对位置），成功率通常越高

结果解读（可选）

  • 日志中应能看到：解析步骤、能力调用顺序、安全校验是否通过

常见问题（可选）

  • Q：解析格式错误？ A：缩短句子、明确物体与动作。
  • Q：执行中断？ A：查看是否目标缺失、断连或安全校验未通过。

6. Blockly 可视化编排

功能目的

用积木表达「识别—判断—移动—抓取—放置」等流程，使逻辑可见、可复盘，适合课堂讲解与考核。

界面布局

  • 工具箱：感知、运动、逻辑、流程控制、安全等分组（以实际产品为准）
  • 画布：拖拽积木并连线
  • 运行与日志：与流程节点对应的执行记录

操作流程

  1. 从工具箱拖出所需积木并按任务顺序连接
  2. 补全必填参数（如目标、坐标来源）
  3. 先跑通简单流程，再增加条件与循环
  4. 对照日志定位错误积木块

注意事项

  • 缺少「识别/坐标」前置时，后续动作往往无法执行
  • 与自然语言模式共用同一引擎时，避免同时抢同一设备资源

7. 视觉识别与工作区标定

功能目的

将相机看到的物体位置转换为机械臂可执行坐标；标定不准会直接导致抓取偏差或安全拦截。

界面布局

  • 视觉结果：类别、置信度、像素坐标等结构化输出
  • 标定参数：由管理员在部署阶段录入或更新
  • 工作区边界：与安全模块联动，用于限制运动范围

操作流程

  1. 在管理员指导下完成或复核相机与工位标定
  2. 运行需要视觉的任务时，先保证光照与目标清晰可见
  3. 若频繁提示识别失败，优先检查标定与摆放，而非反复执行
  4. 发现系统偏差时按运维流程重新标定

注意事项

  • 禁止在未标定或标定失效时强行高速运行
  • 边缘与反光区域误差大，教学任务尽量布置在标定有效区内

常见问题（可选）

  • Q：识别到但抓偏？ A：优先排查标定与工件摆放高度。
  • Q：无目标？ A：检查遮挡、光照与目标是否在视野内。

8. 安全规则与受控执行

功能目的

所有指令在到达机械臂前必须经安全校验；受控执行环境与能力白名单防止脚本越权。

界面布局

  • 安全策略：工作空间边界、速度/姿态限制（以实际配置为准）
  • 白名单：仅允许调用已开放能力（识别、移动、工具、日志等）
  • 拦截提示：越界、断连、非法参数时明确报错

操作流程

  1. 理解「模型输出 ≠ 可直接执行」：须过安全层
  2. 出现拦截时，先读提示，检查坐标与连接状态
  3. 异常后建议先回安全位，再重新开始任务
  4. 课堂演示严禁关闭或绕过安全策略

注意事项

  • 即使有安全层，人员仍须与机械臂保持规定安全距离
  • 对抗性测试表明违规指令应在执行前被拦截，若遇异常请反馈管理员

9. 设备连接与部署方式

功能目的

支持单机与局域网集中部署；控制与图像通道分离，便于课堂稳定与运维。

界面布局

  • 单机：主机上运行服务，浏览器本机访问
  • 局域网：中心服务 + 多工位浏览器（典型机房部署）
  • 状态指示：连接、服务、设备在线状态（以实际界面为准）

操作流程

  1. 按管理员文档顺序启动：服务 → 设备 → 浏览器
  2. 确认端口、防火墙与相机/机械臂 IP
  3. 断连时停止危险动作，恢复后建议先回安全位
  4. 多工位时遵守队列或互斥提示

注意事项

  • 勿在教学中途随意拔插网络或断电
  • ROS/SDK 版本与实验室环境保持一致

10. 故障排除与运维要点

功能目的

帮助用户在不依赖开发文档的情况下，按固定顺序缩小问题范围：界面、网络、设备、标定、安全策略。

界面布局

  • 日志区：错误码、堆栈摘要（若有）
  • 连接状态：设备与服务的在线情况
  • 管理员入口：标定、重启服务、白名单（视权限）

操作流程

  1. 第一步：阅读日志与错误码（对照附录）
  2. 第二步：检查网络与设备连接、相机是否被占用
  3. 第三步：复核标定与工位摆放
  4. 第四步：仍失败则记录截图与复现步骤，联系管理员
  5. 若需性能/安全测试数据，参见项目技术报告（非本手册正文）

注意事项

  • 勿擅自修改未授权配置，以免影响其他工位
  • 同一问题多次复现再提交，便于定位

常见问题（可选）

  • Q：只有本工位异常？ A：对比相邻工位网络与设备编号。
  • Q：偶发超时？ A：注意是否多人同时抓拍/推理导致排队。

11. 附录

术语表

LLM
大语言模型，用于把自然语言任务解析为可执行步骤或脚本。
Blockly
可视化积木编程，本系统中用于流程编排。
闭环
从任务下发到执行与日志回传的完整过程。
能力白名单
任务脚本仅允许调用已开放能力（识别、运动、工具等）。
安全优先约束
物理执行前对坐标与路径进行校验，拦截越界与危险指令。
标定
建立相机像素坐标与机械臂工作空间坐标的映射关系。

错误代码对照（示例）

代码含义
E101目标缺失或视觉结果无效，未进入执行（安全退化）。
E201参数不合法或流程不完整，解析/校验失败。
E301安全校验未通过（越界、危险路径等），指令未下发。
E401设备断连或通信异常，已停止后续动作并提示重连。
